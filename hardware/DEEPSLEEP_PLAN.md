# ディープスリープ撮り溜め — 部材リスト・回路図・設計

現状は CPU/BLE が常時起動で **約200mA → 250mAhで約72分**。これを「**撮る時だけ起きて、
あとは寝る。溜めて、家でまとめてWiFi送信**」に変え、終日駆動を狙う。

```
[起床(RTCタイマ)] → カメラON → 撮影 → microSDへ保存 → カメラOFF
        → (一定枚数 or 家のWiFi検出時) まとめてhubへPOST → 削除
        → ディープスリープ(例:60s) → 繰り返し
```

電池消費の主因は「CPU/BLEが寝ない」こと。ディープスリープで ESP32-S3 を ~µA まで落とす
のが最大の効き目。BLE常時送信もやめ、転送は **WiFiで一括**（高速・電話不要）。

## 2. 回路図

### 2-1. 電源まわり（現構成）
```
   USB-C ──▶ XIAO 充電IC ──▶ 電池(LiPo 3.7V)
                              │
   電池+ ──[スライドSW]──▶ XIAO BAT+      電池- ──▶ XIAO BAT-
                              │
                       XIAO 3V3 LDO ──▶ 3V3レール(MCU/カメラ/SD)
```

### 2-2. カメラ電源ゲート（自作PCBフェーズ・Pch ハイサイド）
カメラの VDD を Pch MOSFET で 3V3 から切り離す。GPIO=Low で ON、スリープ中は
プルアップでゲートHigh→**カメラ完全OFF**（待機電流ゼロ）。
```
        3V3 ───────────┬───────────────┐
                       │ S             │
        CAM_EN(GPIO)──[100Ω]──┤ G  Pch │  ← 100kΩでゲートをプルアップ(既定OFF)
                       │      └────────┤
                   [100kΩ]            D │
                       │               ▼
        3V3 ───────────┘        カメラVDD ──[10µF + 0.1µF]── GND
                                        (+バルク100–470µF を3V3に)

  論理: CAM_EN = Low  → Vgs=-3.3V → FET ON  → カメラ給電
        CAM_EN = Hi-Z → プルアップでVgs=0   → FET OFF → カメラ無給電(スリープ時)
```
> 既存 Sense ボードはカメラ電源がボード一体で取り出せないため、**ゲートは自作PCB向け**。
> 現XIAOでは「ディープスリープでCPU/BLEを落とす」だけでも大きく改善する。

### 2-2b. 推奨：ロードスイッチIC（TCK108AF）で1部品化
ディスクリートFET＋抵抗の代わりに、ロードスイッチICを1個置く（突入抑制＆OFF時放電付き）。
```
   3V3 ──IN          OUT── カメラVDD ──[10µF + 0.1µF]── GND
          │                  （3V3側に バルク100–470µF）
   GND ──GND
   CAM_EN(GPIO)──CTL,   CTL──[100kΩ]──IN   ← プルアップ＝既定OFF(安全側)

  論理(TCK108AF=アクティブLow): CTL=Low → カメラON / CTL=High(スリープ時) → OFF＋自動放電
```
> 発注前にデータシートで**制御極性（Low）と出力放電の有無**を確認。極性が逆の型番なら
> プルアップ→プルダウンへ変更し、GPIO論理を反転するだけ。

### 2-3. 起床トリガ
```
  ・定期撮影: RTCタイマ起床  esp_sleep_enable_timer_wakeup(INTERVAL_US)
  ・任意の即時アップロード(任意): リードSW/ボタン → EXT1起床(RTC GPIO)
```

### 2-4. microSD（Sense搭載・SPI、結線済み）
追加配線不要。撮影/SDとカメラはピンが分離されており共存可。

## 3. ファームウェア設計（状態遷移）

```c
// 擬似コード
void setup() {
  initSD();
  uint32_t epoch = rtc_epoch();           // RTCメモリ保持。WiFi時にNTP補正
  // --- 撮影 ---
  camPowerOn();                            // (PCBではCAM_EN=Low)
  warmup(CAMERA_WARMUP_FRAMES);
  Frame f = capture();
  camPowerOff();                           // (PCBではCAM_EN=Hi-Z)
  sdWrite("/buf/" + epoch + ".jpg", f);
  // （任意）短尺音声/エネルギーVADも /buf/<epoch>.wav に

  rtc_count++;                             // RTC_DATA_ATTR で保持
  // --- 一括アップロード判定 ---
  if (rtc_count % UPLOAD_EVERY == 0 || sdFileCount("/buf") >= UPLOAD_THRESHOLD) {
    if (wifiConnect(HOME_SSID, HOME_PASS, 8000)) {  // 家にいる時だけ繋がる
      ntpSync(&epoch);
      for (file in listDir("/buf")) {
        if (httpPostMultipart(HUB_URL "/jobs", file, kind="store") == 200)
          sdDelete(file);
      }
      wifiOff();
    }                                      // 繋がらなければ即あきらめてスリープ
  }
  // --- 就寝 ---
  esp_sleep_enable_timer_wakeup(INTERVAL_US);   // 例: 60秒
  esp_deep_sleep_start();
}
```

設定（`config.h` か NVS）:
- `HOME_SSID/HOME_PASS` … 家のWiFi（在宅検出を兼ねる）
- `HUB_URL` … **M1のLAN IP**（例 `http://192.168.x.x:8765`）。ESP32はTailscale不可なので
  同一LAN上のhub IPへ直接POST。hubは `0.0.0.0:8765` 待受なのでLANから到達可。
- `INTERVAL`（撮影間隔, 例60s）/ `UPLOAD_EVERY`（例60回＝1時間ごと）/ `UPLOAD_THRESHOLD`

hub側は**変更不要**: 既存 `POST /jobs (multipart image, kind=store/caption)` をそのまま使える。
電話を介さず、グラスが自律で撮り溜め→アップロードする構成へ進化（電話は相棒UI/閲覧に専念）。

## 4. 電力試算（600mAh・撮影間隔60sの例）

| 状態 | 電流(目安) | 時間/サイクル |
|---|---|---|
| 撮影(カメラ+CPU) | ~150–250mA | ~1s |
| ディープスリープ（カメラ非ゲート） | ~5–15mA※ | ~59s |
| ディープスリープ（カメラ**ゲート有**） | ~0.3–1mA | ~59s |
| WiFi一括送信 | ~120–200mA（瞬時ピーク~400mA） | 数秒/時 |

※Sense はスリープ中もカメラ/ボードが通電するため待機が乗る。実値は要実測。

平均電流の概算:
- **カメラ非ゲート**: 撮影デューティ(200mA×1/60≈3.3mA) + スリープ~10mA ≈ **~13mA → 600mAh/13 ≈ 46h**
- **カメラゲート有**: 3.3mA + ~0.5mA ≈ **~4mA → 600mAh/4 ≈ 150h（撮影律速）**

現状(~200mA, 3h)比で **15〜50倍**。WiFi送信ぶんを引いても、非ゲートで**1.5〜2日**、
ゲート有で**数日**が見込める（要実測で確定）。撮影間隔を延ばすほど伸びる。

## 5. 注意点・リスク
- **ブラウンアウト**: カメラ突入＋WiFiピークで電圧降下→リセット。3V3に**バルクコンデンサ**
  （100–470µF）を入れ、**カメラとWiFiを同時に動かさない**（撮影とアップロードを分離）。
- **microSD**: 書込電流/レイテンシ。A1カードを使い、書込はまとめる。
- **時刻**: スリープ跨ぎは RTC_DATA_ATTR で保持＋WiFi接続時にNTP補正。
- **カメラ毎回再init**: 起床ごとに `esp_camera_init`（既存のpower-cycle方式を踏襲）。
- **カメラゲートは現基板では不可**: VDDが基板一体のため。FETゲートは自作PCB前提。
  現XIAOでは「ディープスリープでCPU/BLEを落とす」だけ先に実装して効果を取る。
- **在宅検出**: 家SSIDへの接続可否で代用。失敗は短時間で打ち切りスリープ（消費小）。
